# Performance Enabler — client app shell

A thin consumer shell for a single Performance Enabler app+environment. The runtime SPA itself ships as
the npm package **`@simply-now-enabler/performance-enabler-frontend`** (GitHub Packages); this repo only
pins a version, supplies build-time config, and mounts the app.

This repo is **generated by the platform** (`provisionAppIntegrated` / the self-service provision script).
Don't hand-edit the framework files — platform updates flow in via automated PRs. Your app's content
(`public/{slug}.app.json` + `public/pages/*.json`) is written here by the deploy pipeline.

## Layout

```
.
├── package.json        depends on @simply-now-enabler/performance-enabler-frontend + react/react-dom
├── .npmrc              points the @simply-now-enabler scope at GitHub Packages (auth via NPM_GITHUB_PAT)
├── index.html          mounts /src/main.tsx
├── src/main.tsx        ~20 lines: VITE_* env → RuntimeConfig → createApp(...); imports the package CSS
├── vite.config.ts      builds src/main.tsx → dist/
├── public/
│   ├── staticwebapp.config.json   SPA fallback + *.json exclusion (real 404 for missing pages)
│   ├── {slug}.app.json            app shell — written by the deploy pipeline (do not edit)
│   └── pages/*.json               per-page config — written by the deploy pipeline (do not edit)
└── .github/workflows/azure-static-web-apps.yml   npm ci + vite build → Azure Static Web Apps
```

## Required secrets (set by provisioning)

| Secret | Purpose |
|---|---|
| `VITE_APP_SLUG`, `VITE_DATAVERSE_URL`, `VITE_TENANT_ID`, `VITE_SP_CLIENT_ID`, `VITE_REDIRECT_URI` | Build-time runtime config |
| `NPM_GITHUB_PAT` | `read:packages` PAT — authenticates `npm ci` to GitHub Packages |
| `AZURE_STATIC_WEB_APPS_API_TOKEN` | Azure SWA deploy token |

See the platform docs (`docs/frontend-release-model.md`, `SETUP.md`) for the release + consumer model.
